Resources

Up to 30 resource definitions can be defined, which will be relevant to each order present on the system.
Each resource can be effective at header level – applies to all tasks of the order, at component level – applies to all tasks of the component or at individual task level.

Click in the grid to add or change the resource details. Press the OK button to save any changes to the component header.

Leaving the Process Type column blank makes the resource global to all task of the order.
Marking the Cmp column Y makes the resource common to all tasks of the affected component. This depends on at least one task of the component having a process type that matches the selected process type.

The last column sets default availability for each resource. This is normally N so that each resource has to be set to Y at some point in the life of the order. However for occasional rare resources it may be simplest to default them to Y and when the order first appears in PSA this resource is set to N until the appropriate step have been completed.

Where multiple level approvals have been defined in Radius II you can link each approval level to a resource.
