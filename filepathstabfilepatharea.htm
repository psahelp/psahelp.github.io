File paths tab – file path area

The File Paths Screen allows the relevant data file paths to be configured. All processing is conducted locally on each PC or in a ‘local’ server area under Citrix or Remote Desktop Services.
The ‘Working Directory’ is normally the system installation directory and will contain the local database with other reference files. When a scheduling database is ‘Opened’ from the main file menu a local db is created and populated from the central SQL db and named after the opened database, company and plant enabling all processing to be carried out locally. A second copy of the ‘Live Plan’ is also placed in your working directory and has a ‘chk’ prefix. This copy has two purposes - to enable records deleted from your local plan to be identified and at plan update to identify the field level changes to existing records.
The ‘Reports Directory’ is the location of standard PSA3 reports in A4 and Letter format (indicated by A4_ and Lt_ prefixes).
The ‘Custom Reports Directory’ is the location of modified versions of the standard reports and all other custom reports to be used by PSA3. It may be desired to centralise these in a network location associated with the housekeeping folder.
The ‘CAD Images Directory’ is specialised to one or two customers of PSA and has no functionality for normal use
Use global settings for reports directories – when checked all users see the same paths – recommended
Parallel readers – speeds up distribution of new changes to users – recommended value is 3
‘Use Windows Authentication’ – if this option is selected  and the same user name is used to log on to Windows as to PSA3 then the user will not be required to supply a password and logging on will be automatic. However you must still set up a PSA3 user name and password as this defines the logon mode and runtime options available. As with all applications using ‘Windows Authentication’ it is essential to ensure that screen savers or some other locking mechanism prevents abuse by other users of an ‘open’ screen.
