       Make Ready.                     mr-time                    Job.mrlength

Transferring Item Details

 

The Item Details are read in two stages, Firstly all Item details are read from the pc-item table in the Radius Volume 2 database, their details such as descriptions and customer information is held on the Volumn1 database in tables Ovariety and Customer. This information is collected and transferred to the JobItem table in the Gui Scheduling Database.

 

 

    Company No.            pcitem.kco                JobItem.kco

    Order No.                 pcitem.kjobcode        JobItem.ordernumber

    Component                       pcitem.compno          JobItem.component

    Task Number            pcitem.kprocno          JobItem.Job

Quantity             pcitem.item-quantity   JobItem.quantity

 

Item Code           ovariety.kvar-code             JobItem.ItemCode

    Short Description               ovariety.vardesc        JobItem.ShortDescription

    Description1-5           ovariety.full-desc[5]   JobItem.description1/5       

 

    Cust Short Name               customer.kcussn        JobItem.Cust_ShortName

    Cust Name                customer.name         JobItem.Cust_Name

    Cust Address             customer.addr[4]              JobItem.Cust_Add1/4

    Cust Post Code          customer.postcode             JobItem.Cust_Pcode
