Scheduling

Planned tab actions

Orders may be filtered by work centre or display code, customer, order number like, live/tentative status and delivery date. Filter values are remembered and turned on and off by check boxes. In addition unscheduled orders only may be filtered for and firm orders may be excluded.
A separate filter button refreshes the list after changing settings

 The ability to filter by work centre or display code allows, for example, scheduling of sheet and web order to be completely independent.

Right click a single order gives a menu to schedule, unschedule, unplan or drill down to order detail as on the unplanned tab.

The position of an order in the list is important when more than one order is being scheduled at the same time. A single order may be moved up or down to the desired position but it is not possible to sort an entire column.
The schedule sequence is maintained automatically and is always the sequence in which they are displayed.

By pressing the left mouse button and dragging a number of orders may be highlighted. Right click on this selection for a menu to sort or schedule the group. Please note that the column in which the right click occurs is the column that will be used for the sort.
Typically after moving an order up or down, the whole group between the start and end points should be rescheduled. This makes space on the planning board just where it is needed to resequence the tasks on shared work centres.

When the planned orders are filtered by a work centre or display code it is possible to select ‘Work centre prioritised’ scheduling. This will first schedule the task on the selected work centre and then run a ‘fixed point’ schedule both forward and backwards from this first scheduled task.

The Before and After columns indicate the early/on time/late status based on the user definable colour bands defined on the Preferences tab.

At the bottom of the order list is the utilisation section. This shows the anticipated work load for each monitored work centre over the coming weeks. See Utilisation Monitor above for setup details.

Because of the detailed technical attribute information and utilisation analysis this screen may take some time to build. It is hoped to speed this process up in a future version. Both build and scheduling progress is indicated at the bottom of the screen.
