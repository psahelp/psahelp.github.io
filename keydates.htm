Resources and Key Dates

Key Dates

An enhanced version of resources is available called ‘Key Dates’. This uses the standard resources fields to constrain task start times but also adds links to notices and can be used to create work lists of tasks that are not scheduled but link to the start or end of a scheduled task. It also uses the ‘notes’ area so this is also unavailable when ‘key dates’ are implemented.

Please do not attempt to use this functionality without consulting Radius Solutions.

·      Under configuration/system/functionality turn on key dates

·      Set total ‘global resources’ – recommend 20 – this leaves 10 to be specific for each task

·      Under Tables/resources set default local/customer flags – probably not critical

·      Enter up to 30 standard texts – ones at the top will be easier to find subsequently – they can always be changed later

·      In order maintenance select order and go to Key Dates tab – screen will be blank

·      The first n resources are defined as being the ‘default’ to create

·      Drop down beside Create button allows selection of an existing order as template – the key dates for this order will be copied with the dates adjusted by the difference between the delivery dates of the two orders

·      The Create button populates the table – you still have to ‘Apply’ the changes

 

 

·      1st column is 78 characters free format text describing the date – selected from defaults on drop down or manual entry

·      2nd column is date type – this can be a ‘resource’ on which the task is dependant or it can be a date linked to the ‘start’ or ‘end’ of a chosen task by a given offset

·      3rd column is the linked task. If the type is ‘resource’ this may be left blank and the resource becomes global. However if the type is ‘start’ or ‘end’ then a task must be selected so that a date/time can be calculated

·      4th column is the offset of the ‘start’ or ‘end’ type in hours

·      5th column is expected date – this is the key date so it must be entered for ‘resource’ type dates – it will be calculated for ‘start’ and ‘end’ type dates

·      6th column is date achieved – to be entered later

·      7th column is ‘In Use’ flag – any date can be turned off and turned on again later if required – you cannot delete but can overwrite with completely different details

·      8th and 9th columns flag whether this key date is to appear on ‘local’ (internal) or Customer targeted reports

·      10th to 14th columns are used for linked reports

·      To amend anything click in the box

·      To add a new key date click in the blank bottom row – you must enter text, required date, flags and for ‘start’ and ‘end’ type resources the linked task

·      The list will always be displayed in required date order

·      The time line at the top of the screen displays pointers to key dates – moving the mouse over a pointer temporarily highlights the row below

·      There are two filter options to hide dates that are declared ‘not in use’ and dates that have been achieved

·      Right click a task in the planning board and you can go directly to the ‘key dates’ tab of order maintenance

·      The customer report emulates the current Clowes report – it is expected to be printed on headed paper. Some technical attribute details remain to be resolved

·      The Account executive report list upcoming key dates for all orders via a separate screen that is used to select the executive and date selections to limit the scope of the report.

·      Account executive detail still remain to be imported automatically
