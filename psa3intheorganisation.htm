PSA3 in the organisation

PSA takes data from the Job Costing area of Radius II. It then passes scheduling information to the SFDC system and receives progress updates from SFDC to update the planning board with actual running and completion detail.

In order to maintain the independence of PSA, Radius and SFDC versions these transfers are performed automatically by PsaConnectFor non Radius users the usual method is for the user to populate an Access or SQL Server database that corresponds in format to the Production Control area of Radius 1.40. PsaConnect is then used to transfer data to and from PSA via an ODBC link. This has the advantage that the PSA database is completely independent from the source data and can be upgraded without affecting the source data.

Apart from passing data to SFDC for production PSA also creates a daily log of updates in an Audit file. This currently lists the table, key details, time and user changing records but could be enhanced to provide a full change audit.
