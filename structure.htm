Structure

The component structure of an order relates to the sequence in which components are scheduled are where components can be scheduled in parallel and which components are dependent on others.

Individual components can be ‘dragged and dropped’ around the screen and placed in the required position. Relationship links are then created by using the right mouse button to click each component of a parent-child pair. When the first component is selected the colour will turn white so it is simple to see where links are being created. If the pair already has a link then this action will remove it.

Components left in the ‘Not Related’ area are independent of the other components. When the order is first scheduled these components will start with the other components, but will not be moved during fixed point scheduling operations. Once the required component structure has been produced the ‘Apply’ button should be used to save the changes the planner has made.

The ‘Summary’ window is now visible at all times and displays the relationships as they are placed or removed. These will be created if the 'apply' button is used. The ‘Reset’ button can be used to remove all current component relationships and will place all components back at level 1 in the ‘Not Related’ Area.
