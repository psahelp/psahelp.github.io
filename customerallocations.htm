Customer allocations

Customer allocations allow time to reserved against one or work centre for one or more customers. It assumes a basic weekly allocation cycle but allows for cycles that repeat every 2, 3 or 4 weeks.

For each allocation set up a line in the table with customer, work centre, number of hours (decimal allowed), repeat frequency and start week. The start week is vital if the frequency is not 1 (weekly) as it determines in which weeks the time is allocated. The end week is optional.

In schedule control when a work centre or display code with allocations against is filtered for on the planned tab the utilisation grid will show how much of the allocated time has been used and how much remains available. It will also show how much of the remaining time has been used by customers without individual allocations. This assessment is based on when the tasks are scheduled if the order has been scheduled but are based on the required production week if the order is planned but not scheduled.
