Screen by screen descriptions > Timesheets - SFDC Lite

Set Up

Add shifts – this updates the left hand block of data. Only the No (number) column is critical – the code description, start and end are comment only.

Update status codes (see below for details how to import these from SFDC via PSA Connect). There should be at least one code for types M, G, B and N for Make Ready, Good Copy, Bad Copy and Non Productive. This updates the middle block of data.

Default non-productive code – select one of the available non-productive codes. This code will be used to accumulate time not allocated to tasks or other ‘Non Productive’ events such as maintenance.

Manual time entry is checked if all operators want to enter a time. In this mode the numeric keypad can act as an entry point for the time. If this is not checked, the time is automatically allocated at the moment the operator enters the data. The automatic time allocation will be simpler to operate.

Add users. These user entries must be matched by the same codes in SFDC Maintenance > Operators. The ‘Op code’ field must be entered for all users who will be recording timesheet data.

It is recommended that the majority of users are given an access level of ‘Enquiry Only’. This could be extended to ‘Allow Task Update’ and ‘Allow Resource Update’.

If the Work Centres folder is used, this restricts the user to specific work centres thus minimising their choices when using Timesheet entry. Restricting work centres keeps lists shorter and simpler to use.

For those work centres where machine counters will be used, set the number of digits for the counter (1 - 9). If left at zero, the work centre will not use a counter – see Work Centre Maintenance above.

Option to Import SFDC Stat Codes. This will automatically import the status codes from SFDC. These codes identify the make ready, run and bad  – productive and non-productive (direct & indirect). There must be at least 1 code of each of these types. Note that these codes will initially have been created in Radius II (Company > Costing > Status Codes).
