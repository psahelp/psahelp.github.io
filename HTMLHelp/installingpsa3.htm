Before you start

Installing PSA3

PSA can be run as a local application with many users all accessing a central database or it may be run as a Terminal Services/Citrix application. It may also be run in a mixture of both styles.

The two essentials for any install are that each user has a defined ‘Working Directory’ that is private to them and a PSA3.INI file that is also private – usually in their Windows or Winnt directory. An alternative name and/or location may be specified by defining the full path and file name as a parameter on the command line (protected with double quotes if it contains spaces). This must still retain the INI extension.

If creating custom versions of reports it may be desirable to locate the Custom Reports directory in a central network location. This is fine – just use Configuration/System to point to this directory

When installed locally the default paths will be all that is required. The user will require full permissions in the directory containing the INI file, the Working Directory and the network directory containing the shared Plan.

If installing on a ‘locked down’ PC the process is more complicated – the instructions are

Make a change to the domain policy so that each PC and user has elevated permissions when installing an MSI package. To do this edit the group policy and change: -

 -  Computer Configuration | Administrative Templates | Windows Components | Windows Installer
 -  Enable "Always install with elevated permissions"

      do the same for: -

 -  User Configuration | Administrative Templates | Windows Components | Windows Installer
 -  Enable "Always install with elevated permissions"

Once this has been done reboot the PC and login.  They will now be able to install the software as a restricted user.  Once the software has been installed the Administrator can then undo the changes so that they can't install anymore software.

When installing on Terminal Services/Citrix you must ensure that all servers on the farm have the same version installed, that each user has a single PSA3.INI in a private location – usually $HOME or similar – and this MUST be retained from day to day. In addition the Working Directory = line in this file must point to a private location – usually under $HOME. If the install defaults are used the first user will work with limited functionality – no settings retained – but subsequent users will fail to open the plan receiving permanent ‘file in use’ type messages.

Where there is more than one server in the farm the central database must  NOT reside on one of these servers – total confusion will result as multiple copies are likely to be created and no guarantee given as to which one is accessed at any time. It should be placed on a separate data server
