Special Reports
There are two further custom report s that can be set up.
  • When a task completes PSA can run a ‘completion report’ that acts as a snapshot of details at the moment of completion. The report to run should be defined under work centre maintenance in the ‘Completion Report’ box. If desired this report can be written direct to disk in the background rather than printed. The directory is defined under Configuration/System on the Parameter tab and the file name to be used can contain these variables - %O – order number, %C – component number, %T – task number, %W – work centre, YYYY – year of completion, MM – month of completion, DD – day of completion. The extension should be set to doc, xls or pdf for word, excel or pdf format outputs. Therefore a typical file name might be %O_%T_YYYYMMDD.pdf.
  • The dynamic work list can also output a printed version of the screen. This expects a report named ‘dworklist.rpt’ prefixed ‘A4_’ or Lt_’ for A4 or Letter format. If such a custom report exists a print button becomes available. Contact Radius support for details of fields available to this report
