Before you start > What If and AutoSync modes

Autosync mode

Auto sync mode is for read only and limited update users.

The general rule is that this mode should be used by any user who will not be scheduling orders or changing configuration table details that potentially affect hundreds or thousands of records.

On a technical level it works by taking only one local copy (saving time and space) and resynchronises all changes with the main database approximately every two minutes. It achieves the same effect as taking a second local copy by storing a before image of all records prior to change within the single local copy.

The update process works in exactly the same way – comparing the current data field by field with the before image to determine which fields have been changed and applying these to the central schedule.

However the reload is completely different. There is no longer a full copy of the database - all changes applied by other users since the last synchronisation are copied from the central schedule to the local copy. Typically this will be just a handful of tasks or orders again saving time and space.

Every 24 hours a full copy is performed to take advantage of any major housekeeping clear outs and database compaction that has occurred.

As a result no changes have to be saved and no reloads are required – in fact the update menu options no longer appear. Only change details are passed keeping network traffic to a minimum. Any changes made by the user take a maximum of 4 minutes to reach all other auto sync users.

Users can be limited to either mode of operation if required. Within the user maintenance screen the allowed modes can be defined and if both modes are allowed a default can be selected.

To switch between modes click the Mode indicator at the bottom right of the main menu screen – it is recommended that mode switching is done before a schedule is opened.
