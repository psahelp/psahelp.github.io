Technical Attributes

 Under Radius there are 20 defined within the estimate and applying at the header level. If the estimate changed there was no guarantee that the technical attributes would be changed – in fact the opposite was likely.
At the present time this functionality continues – the new features add to and do not replace the current system. It is likely that for Radius II users the estimate defined technical attributes will no longer be supported.

The new technical attributes are held at task level – essential for any order where different tasks operate on different materials.

Almost any information can be specified as a technical attribute and will be copied from Radius II source data. The primary source files are the works instruction and item definition tables. Likely candidates are sheet or web and item sizes and weights, inks, gluing styles etc.

As the information required is defined under Configuration/Tables on the Technical attribute tab no separate estimate entry required and changes to the estimate and recreation of the order result in new, correct information being transferred.

The actual pick up of information is by PsaConnect as the order is transferred.

Header level attributes are still available and work as before except that the maintenance of attributes has been improved with natural ‘in cell’ editing.

At task level you will see and maintain task specific attributes if present. If not present you will still see and maintain the header level attributes.
