Installation and set up

PsaConnect

Turned on in PsaConnect – this triggers 2 new tables in PSA (1 for tool specifications, 1 for physical tools) to be populated and kept in sync with the Radius tooling library.

PSA

Because of multiple tool types – including user defined types – all the tool attributes (sizes, types, gaps, shape etc etc etc) are imported as attributes allowing complete flexibility in PSA

The left part of the screen should show all tool types – some relate to older versions of Radius (Estimate Die, Coater Plate, Embossing Tool etc) and some should be user defined tools set up within Radius – run the PsaConnect Special Transfer ‘Populate Attribute Field Lists’ if these are missing or new tool types have been defined.

Next select the attribute details to import. If the attribute area is empty the simplest way is to press the ‘Default’ button.

For some tool types PSA will populate a default attribute selection – if none is known then the default source for each attribute must be selected in the same way as normal technical attributes by clicking in the source column and selecting a field from the drop down – then giving this field a title to be used within PSA in the description column.

Once selected it is best not to change the attribute number allocated – but you can change the sequence fields are displayed in by clicking in the Attribute column then moving up or down – as above where approval field have been moved to the end of the list.

Apply the changes and update the plan.

Tool statuses are user defined in Radius and there is no way to define when a tool is available or not – this is handled by the display bottom right – to save time default values are set up but these will probably need to be changed as up to 8 bands/colours are defined – you need to group all the possible statuses into one of these bands.

Multiple statuses in the same band should be comma separated with additional leading and trailing commas as shown.

For improved prediction of tool required dates a tool type can be associated with a process type.

Currently this is manual entry – both columns should really be selected from a list – work in progress.
