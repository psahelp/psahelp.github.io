Installing and configuring PsaConnect
PsaConnect is usually installed on a server so that it can be close to Radius, SFDC and PSA databases. Only a single instance is required for a single plant. If multiple plants exist it is essential to run separate instances for each plant – only one install is required and multiple runtime instances can be started. Create a desktop icon for each with a command line pointing to the executable and having a parameter pointing to and INI file for this instance of PsaConnect. The command line should then look like
C:\program files\psaconnect\PSACON8.exe “C:\Windows\ConnectMyFirstPlant.INI”
