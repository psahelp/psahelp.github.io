Non Working Periods

Non-working periods or ‘shifts’ can be represented on the system so as to provide an accurate factory working plan. Tasks that are placed on the planning board either manually or through the scheduling routines will behave differently depending upon which ‘shift method’ has been selected. This is configured locally under Configuration/System on the parameters tab.

Note:- Shift Data should be added well in advance of scheduling orders, it is recommended a shift pattern is created for a single week, this pattern can then be extended into future weeks periodically using the ‘extend shifts’ facility.

User defined colour and comments may now be added to individual non working periods in the New or Edit routines. The comments will show as Tool Tip Text and give a reason for the non working period.

·    Shift Method 1 – Start task after non working period

If ‘shift method 1’ is selected then tasks encountering a shift will be automatically placed at the end of the shift, if there is sufficient time for its ‘run’ before the start of any following shifts. The task will continue to be moved forward in time until there is enough time for the run to complete.

·    Shift Method 2 – Stretch task over non working period

If ‘shift method 2’ is selected then tasks encountering a shift will be automatically adjusted so as to accommodate the ‘down-time’ into the task run length, making the task appear to ‘stretch‘ over the shift period.

·    Maintaining Shifts

Shifts can be maintained from the planning board using the ‘shift maintenance’ menu. Alternatively ‘shift maintenance mode’ can be activated by pressing the  button. This causes all existing shift periods to be displayed.

If the  button is toggled then the two adjacent buttons will become enabled. These are ‘New Shift’ and ‘Extend Shifts’, both of which are available from the ‘shift maintenance’ menu.

·    Adding New Shifts

New non-working periods or shifts can be added by clicking on the  button which will activate the ‘New Shift’ screen.

The relevant work centre should be selected from the combo box together with the duration of the shift in hours and minutes, the required start date and time must also be entered. On pressing the ‘Apply’ or ‘OK’ buttons the system will create the shift after first checking that the new shift period does not overlap any existing ones.

·    Extending Shift Patterns

As mentioned earlier it is important that shift data is created and in place before orders are actually scheduled over a given period so that individual tasks can be positioned to take into account any non-working periods.
The ‘Extend Shifts’ facility allows a selected weekly shift pattern to be extended across all available work centres and forward in time.
A typical week’s shifts pattern can be set up for the topmost work centre; this can then be extended across all work centres if required. Work centre specific changes can then be made using the ‘Shift Edit’ facility (see later) where individual shifts can be deleted or their parameters changed.

·    Editing Shifts

Individual shift periods can be deleted or adjusted using the ‘Shift Edit’ screen; this is activated automatically by double clicking on individual shifts.
The shift periods start and end date and times can be adjusted to the nearest minute, the minimum shift duration is 30 minutes.
