Converting the database
You need to run program PSA_Convertor (from the PSA3 install directory) – this requires
  • A database to convert – browse to your current PSH database
  • Path for housekeeping files on new database – this can be the path of the PSH file or the physical location of the SQL database created above – the adt files will be stored here
  • The server – ANDREWS-E5510\SQLEXPRESS from the example above
  • The name of the target database – GPI from the example above
  • If using ‘Windows Authentication’ the user and password can be left blank
Now press ‘Load Plant’ – progress will be reported at the bottom of the screen. The longest step will be ‘Load the data’ when the progress bar will indicate progress. If converting a second or subsequent database you will get the prompt ‘Tables exist replace Y/N?’ to which you should respond No so that data from the second plant is added to the first.
At the end of this process the original PSH database remains unchanged so can continue to be used by PSA2.
