Menu Options

The Menus now include virtually all options available from the main menus screen so it should become unnecessary to return to the main menu under normal circumstances. Those described here are not available from the main Menu Screen.

File – Display Code – Re-opens the Planning Board Preferences screen

File – Start Date – opens a calendar to set a new date for the left side of the planning board

File – Print – opens a printer dialogue to enable a print of the entire planning board to be sent to a printer – the screen is sized for best fit on the standard paper

File – Reload Plan – will abandon any unsaved changes and load a full version of the plan. If there are unsaved changes a warning is given

File – Align Planning Board – Opens a separate screen where the planner can manually align data returned from the Shop Floor. It is now normal to align this data automatically using transfer options In PsaConnect.

The planner is able to update the scheduled task data with running or completed job information from the ‘SFDC’ system and can be completely selective as to which jobs are updated.

The ‘Select’ button can be used to list all running tasks as specified by the ‘Include’ check boxes. Tasks can be de-selected by double clicking on the task to be removed in the grid. By checking the 'Current Display Code' box only tasks for work centres in the current Display Code will be shown.

There are two different update styles available and these can be selected at the time of update by using the combo box at the bottom of the ‘Planning Board Update’ screen.

If the ‘Mapped to SF Times’ update style is selected the tasks will be updated so that their scheduled start time matches the actual start time of the corresponding running job on the shop floor. Similarly the task’s scheduled end time will be updated to match the calculated end time, if the job is still running, or the actual end time if the job has been completed.

If the ‘Moving Start Times’ update style is selected then the planner can enter a start date and time at the bottom of the ‘Planning Board Update’ screen to which all the tasks scheduled start times will be set. This only applies to jobs that are currently running. If a job has been completed then the task’s scheduled start time will be updated so as to match the actual start time of the completed job. The task’s scheduled end time will be updated so as to match the calculated end or actual end time if completed.

Any tasks that are unable to be updated are listed in the ‘Exceptions List’. This may occur when using the ‘Moving Start Times’ mode if a start time is entered which may start after some of the jobs have actually ended. The Exceptions list can be viewed by clicking on the ‘Excepts’ button.

File – Exit – Closes PSA3 completely – you will be prompted to save any changes

Display – Attributes See details under the Technical Attribute Display below.

Display – Order Colour – resets the planning board display to the standard order colours

Display – Order Details – opens the Order Maintenance screen

Display – Replot – replots the planning board for the current horizon. May be required if normal checks for replotting have been turned off

Display – Spotlight – see the Spotlight Facility below

Display – SFDC Options – this opens a screen where you can select which SFDC tasks running bars are shown for. At least one of ‘’Running tasks’, ‘Suspended tasks’ and ‘Completed tasks’ must be selected – it is normal to select running and suspended tasks.

If the SFDC refresh option is turned on then the refresh frequency can be selected here. Note that having the SFDC refresh turned on will update running details on the planning board whether or not there are unsaved changes.

Schedule – Assessment Check – this runs a full assessment check on every planned order in the system. It is only normally required after a major change such as changing the process type of a work centre. Depending on number of orders it may take some time to run.
