Availability

Repeating downtime periods such as weekends should be added in the Availability tab. As many distinct patterns are required as exist across the work centre of the plant. Each pattern is defined by pink block that indicate when the work centre is AVAILABLE. Corresponding non working period will be set up to indicate when the work centre is NOT AVAILABLE.

A typical example
Click ‘New’ bottom left and enter a code and description.
Choose a date for your week to start – normally Monday of the current week
Specify how many days until the period repeats – usually 7 but up to 28 is allowed

Enter how far in the future the shifts will extend – don’t extend further than necessary – it is kept up to date automatically and extra days increase database size and reduce performance – typically 30 – 90 days

Use the ‘new’ button above the timeline to add a block. Stretch and drag it to the start and end of shifts. You may need to add extra blocks to define the ‘shape’ you require. It doesn’t matter if blocks overlap. You can micro adjust the start and end points using the adjustment arrows above.

Select the work centres this pattern applies to. A work centre can only be linked to one pattern – if you link it to a new pattern it will be unlinked from the previous pattern.

The pattern will be repeated until the necessary days have been applied. Shifts will automatically copy into the future and remove periods in the past.

The daily ‘housekeeping’ process will check all patterns to see if they need to extended for a further ‘repeat period’ and the ‘next start date’ will advance.

If you want to change the pattern say from the start of next month just change the pattern and reset the ‘next start date’ to the date you want it to apply from
