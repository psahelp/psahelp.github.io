 Question : When I do a forward schedule the orders are planned but starting a week ahead? ‎(1)

 

There is a parameter under Configuration/System/Resources called 'Days forward to schedule tasks with unavailable resources'.  There is also a scheduling parameter on the preferences tab Resource constrained scheduling.
With the first parameter set to 10 days and the second checked, all tasks with unavailable resources schedule at least 10 days into the future.  The most common method of scheduling is to not set these parameters but have someone check the board and make any unavailable resources available for the scheduled time.

 Question : How do I understand the details in the audit file? ‎(1)

 

Audit files are named uniquely for the day they were created and including the name of the database.  A new file is created each day with all records appended to the file in chronological order.

 

Lines include date and time information and program version.  More recently, the record type information has been added to each line, these are:

- PM - Progress Message from PsaConnect.

- FP - Data transferred from Radius.

- TP - Data transferred to Radius.

- FS - Data transferred from SFDC.

- TS - Data transferred to SFDC.

- IN - Information.

- QQ - Queue control information.

- ER - Errors.

- SF - tasks deleted from SFDC.

- TD - updated records from PSA.

- ZZ - Used by PsaConnect for firm line corrects.

- CN - SQL synchronisation details (PSA3 only).

 

Queuing Control information lines include information on what the queue user wants to do, these are: J - a request to update, T - test for permission, P - continued progress, D - update complete.

The lines also contain information on whether permission is given, where the request came from, an internal reference number and the current plan update number.

 

Update Records hold information on the upstatus value of records being written, the upstatus value of the DB opened, the user updating, the table being updated, the key of the table being updated and the action being taken: A - Add, C - Change, D - Delete.

 

For PSA3, STRTIME has been altered but this still shows the old values.  It also holds information on the type of user (what-if or auto sync) and also the last SQL statement generated.

 

Finally there is a useful tool for PSA supervisor users which allows them to view the audit log and allows filtering on the details.  These can then be saved and sent to Radius for analysis.

 Question : How do I upgrade PSA3 and PSAConnect? ‎(1)

 

1. To upgrade PSAConnect all you need to do is replace the current .exe file with the new .exe file you have received.  Before you can do this, the currently-running PSAConnect must be closed.  Once copied, restart PSAConnect.

2. If you are using the auto-loader for PSA this makes upgrading more straight-forward:

Essentially, you need to put the contents of the zip file you received (psa3.exe, psa3.chm, build.txt) into the same dir as the loader. Nothing else needs to change. If they want to kick users off dynamically to replace the .exe then rename the stop.txtx file to stop.txt. Once the changes have been made to the .exe files, change the name back to stop.txtx

If you are not using the auto-loader, each installation will need to be upgraded.

There are additional instructions in the appendix in the manual.

 Question : How do we get Jobs from Radius automatically into the Planned window in Plan Control in PSA? ‎(1)

 

On the Transfer Options window, check the 'Import Orders as Planned' option.  Note that if there is an amendment, then the Job will always go to the Unplanned window.

 Question : How do we set up users as read only so they can not make updates to the plan? ‎(1)

 

Set these users up with Access Level 4 - Enquiry Only - this prevents updating.

 Question : How do you create separate databases for PSA? ‎(1)

 

Use one of your existing plans and under Utility/New Plan enter the new plan name. The default selection of bits to retain is probably right  you may want to add users. It only takes a second or two to create  then open the new plan in its own right and under Configuration/System/Parameters set the correct company and plant.

 Question : How do you make a user able to update the schedule? ‎(1)

 

Give the user full access to the database by giving them supervisor permissions and setting them to a 'what-if' user on the second tab of 'User Maintenance'.

 Question : I have created a new display code but other users cant see it? ‎(1)

 

The planning board preferences drop down is populated when you first go in - so check the option 'single order' then recheck the option 'display code' and it will pick up new display codes.

 Question : If a job is out of the firm zone, will it go through to SFDC? ‎(1)

 

Answer: No.  If defined as 'Isolated Firm Zone' then the Firm Line cannot be moved but the tasks can be dragged above it.  If not defined as 'Isolated Firm Zone' then the Firm Line can be dragged on the Dynamic Work List as it can on the planning board.  It prevents tasks going to SFDC if the WC SFDC transfer option is set to 'When in Firm Zone'.

 Question : If an order disappears from PSA without being produced, how do you get it back into PSA. ‎(1)

 

Use 'Radius Single Order Import' in the 'Special Transfers' Tab in PSA Connect.

 Question : Is there any way to auto-start PsaConnect if the server it runs from is automatically shut down? ‎(1)

 

The best way is to put an entry into the server startup script to launch PsaConnect  something like

C:\.....\psaCon8.exe c:\windows\<my>psacon8.ini

This will start PsaConnect  and if you have checked the option save and restore on close then it will start running automatically if it was running when closed down.

There are further options on the 'Databases' tab which allow you to enter a time range whereby PSA Connect will stay idle.

 Question : Job shows (in Radius) as completed in PSA.  Re-import does not work nor does 'Single Order Import'.  How to get this back into PSA? ‎(1)

 

Reset production status on the Job to not started.  Then carry out 'Single Order Import'.

 Question : PSA Connect fails due to a Radius/SFDC record lock. ‎(1)

 

In Radius from the menu select Help -> About -> DB Details -> Locks.  Change the database to Radius or SFDC depending on the lock type and you will see who is locking the records.  You can then clear the message and the locks should be removed.  Then PSA Connect will resume.

 Question : Updated job status to 'complete' in PSA but it reverts back to 'planned'? ‎(1)

 

In general, SFDC dictates what happens on the shop floor and so overrides PSA with regards to job status.  One exception is when a job has been suspended - in this case it should be marked as 'complete' in PSA and to make sure the information gets to SFDC there is a configuration option in PSAConnect that should be set.  The option is 'Update SFDC to Complete' and can be found on the 'Transfer Options' tab.

 Question : What affects the size of the PSA Database? ‎(1)

 

The size depends on the number of jobs, complexity of jobs, how long completed jobs are retained (user defined) and non working periods (particularly how complex and how far into the future). Also, how far ahead non working periods are defined has a big influence on size - there is no point in laying down 12 months if there are no jobs beyond 2 months.  So size can vary between about 5 Mb and 150Mb - a compact & repair will give the true size and should be run regularly on larger databases.

 Question : What does part scheduled mean in PSA? ‎(1)

 

Part Scheduled implies some of the tasks on the job have been scheduled and some are still unscheduled.

 Question : What field in the Radius database flags the job as transferred to production scheduling?  ‎(1)

 

There is a field in PV_Job called PSAextracted that is set to True once the job is transferred.  Until then it is set as False.

 Question : What option needs to be set to allow for auto-planning which will then push jobs through to SFDC? ‎(1)

 

There are 2 ways of doing this.
You need to have import as planned turned on.
To make them pass directly to SFDC you must either
1) Set the transfer option in PSA work centre maintenance to the second option  before scheduled or
2) In PSA under Configuration/System/AutoMerge set the option automatically schedule new orders imported as planned.

 Question : When a user tries opening a report in PSA, they receive 'Logon Failed' error. ‎(1)

 

Report files may be out of date - try updating with up-to-date versions.

 Question : When does the First Call Off Date field get populated in PSA? ‎(1)

 

Depending on which settings are selected on the Transfer Options tab in PsaConnect ('Import Item First Call Off' and/or 'At Order Level Only'), the First Call Off Date is set to the Ship Date from the Open Shipping Request at Item Level and/or Sales Order Level.

 

Because it can take a long time to run through, it runs about every 2 hours, not with every transfer.  It can populate the First Call Off Date for that Item (for possibly several jobs) or just for that Item on that specific Job (in which case there may be no date at all).

 Question : When try to create a custom report through PSA I recevied an error and the report is not created. ‎(1)

 

If the error is 'Logon Failed' then the DSN in question is pointing to a broker that isn't valid from this location or the broker is not running at all. 

 

If the error is 'Failed to open connection' then the DSN does not exist. 

 Question : Where do I specify the length of audit history - how much should I keep? ‎(1)

 

20 - 30 days is recommended.

 Question : Which field to add to the technical attributes table in scheduling that will bring in the adhesive? ‎(1)

 

There seem to be 2 distinct ways of handling this in estimating one results in the adhesive being treated as a lamination layer and the detail on wi-lam  the other treats adhesive as a coating and the detail ends up on wi-rcoat.

To find which is in use we suggest they go to PsaConnect/Utility/Database View (pwd aval0n numeric zero), select Radius from the database dropdown, replace the select... statement with each of the examples below replacing n with the correct company number and xxx with a suitable job to test and then query

select * from [wi-lam] where kco = n and korder = xxx

select * from [wi-rcoat] where kco = n and korder = xxx'

 

(Make sure you have run a Populate attribute field list from PsaConnect Special Transfers tab to get the latest into PSA.)
