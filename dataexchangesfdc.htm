Data Exchange SFDC

PSA to SFDC

The transfer to SFDC involves populating the sfplan table. Depending on user options, the PSA firm line, live/tentative status and planning status task may also be removed from sfplan. Processing has to ensure that transfers to SFDC can be triggered solely by the passage of time as tasks drop into the planning board firm zone in PSA with no change required in PSA.

Where tasks have been split in PSA (recognised only by PSA and SFDC) the table sfjobitem is also updated so that SFDC knows how the task has been split into separate sheets/items.

SFDC to PSA

SFDC data has remained fairly stable and unchanging between Radius I and Radius II. Almost all processing is at task level rather than job. At some sites there is data transfer at print change level.

The primary source of data is table sfjobdet and is triggered by the changeflag field – ‘A’ for running tasks and ‘C’ for tasks that have been completed or suspended since the last transfer. The flag on running tasks is left unchanged as SFDC is updating it every few seconds. On suspended and completed tasks the flag is set to ‘U’.
