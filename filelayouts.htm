Appendices including technical notes

File Layouts

The key data tables are ordheader, component, job. These hold all the necessary details for the order. Job holds the task details. Jobitem and jobmaterial hold optional additional information.

Comp_relationship links components and provides the structure to the order together with the sequence of tasks within a component.

Resource holds work centre data. Any number of work centres may be linked for display and reporting purposes by a displaycode. A group of work centre can also be created for scheduling purposes by having the same value for capability. A work centre can only belong to one capability group. Within the capability group a work centre may be further defined by technical attributes. This is held in the TechMap table.

The customer, password, parameter, passwordresource, techattributes and processtype tables contain static data. Opevent and eventcds hold data intended for SFDC and costing input.

The SFDCdelete table holds a temporary list of orders or tasks deleted from ordheader etc that are used by PsaConnect to delete records from SFPLAN in SFDC.

The four report tables only contain temporary data while reports are processed.
