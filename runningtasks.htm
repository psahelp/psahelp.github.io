Running tasks

Press the Tasks button and a list of tasks is displayed – highlight one and press ‘Add’. If there is only one defined Make Ready status the task will immediately go into Make Ready with this status.

If there is more than one then the correct status must be selected from the list. Note that the description appears in the Reason box.

Time is started automatically or entered (if manual mode is set for the whole system). Note that all the data relating to the job that is being worked on, is displayed in the upper part of the screen

At any time a quantity can be entered (400 in the example below), press OK and the entered quantity will overwrite the quantity previously entered against the operation code. If the work centre uses a Counter (entered against the work centre maintenance – see above), the entered quantity will be the one logged by the counter and the difference between the current and previous entry will be calculated.

When the status changes e.g. make ready to good copy, the time is constantly being recorded or will be entered (if manual mode is set for the whole system) for the next operation. Select Good or Bad (as appropriate) and the drop down is displayed if there is more than 1 code (Run and Running in the sample below). Click on the correct one. If there is only 1 relevant status code for the selected operation, there is no need for further operator selection.

The operator will record the current status of the job by moving between Make Ready, Good Copy or Bad Copy entering appropriate quantities at each stage

When a task has been suspended or completed, enter the quantity appropriate to the previous operation e.g. MR, good copy or bad copy, and select Suspend or Complete. ‘Suspend’ is equivalent to ‘lifting’ the job and if further work is done on this task again, the task will have to be assigned to a user and a work centre as if starting it for the first time. ‘Complete’ means that everything necessary for this task has been finished.

After Suspend and Complete, all data previously recorded on this shift is now in a state to be transferred. In practice this will be controlled by the supervisor.

When an operator ends work for the day:-

‘Suspend’ or ‘Complete’ all tasks and select Logoff – this has the effect of ending the shift and allows all the logoff events to be closed and made available for transfer.

If the user attempts to logoff before suspending or completing tasks, the user will be asked to confirm their action.

All operators should Log off at the end of their shift.
