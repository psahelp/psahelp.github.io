Before you start

Essential parameter check list

This is a check list of the steps to install, configure and set up a multi-user PSA system

·      Install a single instance of PSA and log on using User1/123456

·      Convert the Sample3.psh database using the PSA_Convertor utility

·      From the Utility/New Database create a new database in a data server location – retain nothing

·      Open this new plan and go to Configuration/System

·      On the File Paths tab set Paths, Housekeeping and local parameters – see individual screen for details

·      On the Functionality tab set the required options – see individual screen for details

·      On the Parameters tab set company details, system parameters, calendar and analysis set details – see individual screen for details

·      Also set up Resources and Warnings details - see individual screen for details

·      On the materials tab run the ‘Restore’ option and if material are going to be declared critical at sub group level check the relevant box - see individual screen for details

·      Apply, close and Update these settings

·      Install PsaConnect – preferably in a server location – ODBC data sources are no longer required – all connection details are entered for Radius and SFDC on the Database tab.

·      Start PsaConnect and link to the PSA database – should display company and plant

·      Link to Radius (and SFDC if installed) and check connections.

·      On the special transfers tab run the Import Work Centres, Import Material Groups and Populate Attribute Field List options

·      Back in PSA update again to draw the new details into your local copy.

·      Under Configuration/System on the material tab mark the groups/subgroups that are going to be critical – these must have requirements raised that will ultimately be satisfied – all other materials (typically inks and some packing materials) will be deemed available at all times. Update the changes.

·      Under Configuration/Tables Process types and default sequences should be set up for each process to be scheduled. The appendices contain a list of process types used in Radius II but you can define your own. Typical sequence numbers might be Print-10, CutCrease-20, and Finish-30 with other processes fitting between as required.

·      Under Configuration/Tables work centres have been imported with only very basic details. Before proceeding set up for each work centre the process type (and hence process sequence), the default style, default speed, default make ready, minimum run time. Set cost rate to zero.

·      Remove any work centres that are not going to be scheduled to keep things tidy.

·      Decide on some resources – in common usage is ‘Plates Ready’, ‘Ink Ready’, ‘Artwork Approved’, and ‘Outers Ordered’. The list will grow.

·      Decide on some technical attributes – these have the disadvantage that they are only populated during transfer from Radius so adding new ones later will only apply to new or re-imported orders after that date.

·      Under Configuration/Tables create at least one display code – the system expects at least one but this cannot be created until after work centres are imported. It can be deleted or changed later once display codes are set up for individual users or departments.

·      Import a couple of sample orders

·      In Schedule Control define a basic set of columns to appear on the unplanned and planned tabs using the check lists on the Preferences tab

·      Set up default ‘task ticket’, ‘pop up task card’ and status bar displays for task details to be shown at run time.
