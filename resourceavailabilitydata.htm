Resource Availability Data

 

The resource availability information is held against the order header table in the Radius system. The same information is stored at component level in the Gui Scheduling System. Therefore during the import process, the resource data is read from the pcjob table and stored temporarily in arrays to be tranferred to the relevant order components.

The following fields are array types indexing from 1 to 10. Each index corresponds to a Fixed Resource in Gui Scheduling defined in the Component Type Table, which appear against all components.The pcjob.res-compno[10] field contains the number of the component the resource data is relevant to.

Field Information

 

pcjob.res-compno[10]   :Holds the component numbers for which the corresponding resource data (of the same index number) is to be allocated.

          ~Gui Scheduling DB Component.Compnumber     

 

pcjob.res-stat[10]        :Resource[n] Available Y/N

          ~ Gui Scheduling DB Component.Resav1-10

 

pcjob.res-avail[10]               :Resource Available Date

          ~Gui Scheduling DB Component.Resavdate1/10

 

pcjob.res-procno[10]            :Resource Dependant Process

          ~Gui Scheduling DB Component.Processtype_1/10

 

 

 

Note: The number of resources in Gui Scheduling has now been extended from 10 to 30, in release 211.    Only the first 10 however are populated from Radius.
