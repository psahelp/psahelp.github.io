Utilisation Monitor

Depending on the Preference tab setting for ‘Monitor on Separate tab’ the Utilisation details are either below the Planned grid on the Planned tab or on a separate Utilisation tab.

To enable the Utilisation Summary either the Planned tab must be filtered by a work centre or display code or the same must be selected on the Preferences tab

The original design was to show available and utilised hours each production week. However the design has been enhanced to allow assessment by calendar month, production week, production day or production shift.

Shifts are assumed to last 8 hours but you may adjust the start time of the first shift each day by a positive or negative amount. If the first shift Monday actually starts at 10 p.m. on Sunday use an offset of -2.0 hours. If the first shift Monday starts at 6 a.m. Monday use an offset of +6.0 hours. These offsets also apply to all other periods.

The Preference tab option ‘Assess all task time in first week’ only has an effect where a task is scheduled to span a week. It is assumed it must complete in the initial week. This might not be valid where very long running tasks are involved.

Times from task not completed in previous periods are always rolled into the current period – often overloading it when it will turn pink. The length of the current period is always reducing as the calculation is from now.

In addition to assessing the time in each period the first material usage is also estimated – this can only be relied upon for press tasks but can provide a useful guide to daily material requirements where long running tasks are involved.

A Print button allows the grid to be printed in a more report friendly format.
