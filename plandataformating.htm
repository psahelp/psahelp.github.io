Plan Data Formating

This section opens up a number of utilities that may be required occasionally – always use with care and be prepared to discard any changes by reloading the current plan if things go wrong

·    Date Advance

Strictly for testing and development only. When a plan is not being maintained regularly it quickly goes ‘out of date’. However just shunting tasks to the time line solves the missing task issue but leaves all orders late. This option allows every date in the system (schedule, delivery, promise, resource expected and achieved etc) to be advanced by +/- n days. For non working periods to look right it generally has to be in increments of +/- 7 days. As stated above – strictly testing or development only

·    Shift Overlap Correction

Although it should not now be possible to create shifts or non-working period which overlap one another this utility when run will remove any overlaps automatically. Not normally required – overlaps are now well handled in the day to day processes

·    Database TidyUp

This utility checks the order structure from header through components, tasks, items and materials removing any records which are not correctly linked to a higher level record – orphaned records. The Live Plan should be updated after running this utility. Because of the way materials are updated it is quite conmmon for material records to be orphaned in this way.

Two new option allows components without tasks and orders without tasks to be removed. As these are effectively redundant the scheduling process will be speeded up without any loss of functionality.
