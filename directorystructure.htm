Reports

Directory structure

During installation a PSA3Reports directory is created within the install directory and containing all the standard reports. These should not normally be changed or renamed. It will also create a Custom directory with the PSA3Reports directory to hold your custom reports. However if custom reports are being created this can cause a distribution problem, both when install on many individual PCs and when installed on more than one Citrix server, so you may wish to create a separate report structure – possibly below the directory used to hold the central .PSH database. Once you have pointed users to the correct location (Configuration/System/File Paths) you will only have to maintain a single copy of each report.
