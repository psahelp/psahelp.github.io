Standard Reports

This is a brief description of the standard reports. Most of them display the Crystal file name in the bottom left corner. To find a report to make a custom version of add a prefix (A4_ or Lt_) and a .rpt extension to the names below. By far the most used report is worklist – it contains many options and subreports for both items and materials

The scheduling assistant has a number of reports available. These are accessed from the main or planning board menus. The ‘Order Information’ allows the planner to run a number of order based reports, described earlier in the ‘Order Maintenance’ section.

The ‘Schedule Reports’ menu option provides access to a number of schedule query reports.

A typical report window allows the user to move through each page by using the arrow buttons at the top of the form, similarly zooming in and out can be achieved by using the drop down % box.

A print out can be obtained by pressing the print button. Alternatively the report can be saved to a text or other file format by using the ‘export’ button. Multiple Export formats are supported including Word, Access and Excel. They may also be e-mailed directly to recipients.

Reports are in Crystal Reports 9.0 format and may be modified by users.

Report headings are automatically printed in the language of the user as defined in system configuration. The paper format (A4 or Letter) can also be selected in system configuration.

A custom version of  any standard report may be created and will be used by default. It should, for example, be named 'A4_worklist.rpt' or 'Lt_worklist.rpt' depending on paper format and should be put in the 'Custom Reports' directory. This version of the report will be used if it exists. It is thus possible to create different versions for each user. If basing your custom worklist on one of the standard worklist reports you should remove the sub-report as it will not be initialised by PSA
