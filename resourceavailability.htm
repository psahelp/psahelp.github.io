Resource Availability

Details of resource information maintained against individual orders can be obtained using this report. In order to produce each component of a specific order the necessary resources must be available. The resources can be linked to specific tasks within each component. These are called dependant processes.

The report will provide details of the resource availability information which affects the production of individual components, any dependant processes together with their scheduled start times are also detailed, hence the scheduled start times of dependant processes actually equate to the resource required dates.

The user can select which resources he/she is interested in, and resources required over a specific date range can also be requested to reduce the volume of data returned. The report can be sorted in Order/Component order, or in resource order. Only resources linked to specific dependant processes and in turn specific work centres can be specified.

A new field Format has been added with options ‘Standard’ and ‘With Notes’

If the sort option ‘Resource’ is selected this will now sort by Resource and Required Date/Time ascending. Thus the resources that are required soonest will appear at the top.

There has been some improvement to the navigation of the screen for users selecting fields by mouse.

A new field ‘Print Component Number’ is added to allow the printed Component description to include or exclude the Component number. Where users have their own component numbering system printing the PSA value could cause confusion.

All entered values except actual display code, work centre and dates are remembered from session to session.
