Resources

Report Resources – If checked, resource information having blank entries will not appear in reports.

Utilise Resource processing – these options are no longer available. Resource configuration is explained in the ‘Using the new features’ section.

Materials may be excluded from resource availability calculations - if checked then material data is treated as read only and the remaining flags have no meaning. If not checked then a separate materials tab is available

Last Delivery Time – If Checked, A last resource delivery time can be provided which will be used in conjunction with all resource delivery dates when performing ‘Resource Constrained Scheduling’...

Dynamic Work List Options

‘Do not allow scheduling of planned tasks on the dynamic work list – normally checked – the Dynamic Work List shows the running task at the top followed by the scheduled tasks in sequence followed by planned but unscheduled tasks. Moving one of these planned tasks into the scheduled area will schedule it. These results in an order with some scheduled and some unscheduled tasks – beware.

‘Do not lock running task at top of Dynamic Work List’ – normally unchecked – by definition a running task must be running now and all other tasks can only be scheduled after the running task. Therefore the running task is normally locked at the top of the Dynamic Work List. However for technical reasons it is occasionally desirable to be able to change the sequence and temporarily checking this option allows problems to be cleared.

 ‘When updating attributes at task level prompt for task only update’ – normally unchecked - attributes are held at task level but normally the first valid task attribute is copied to the header level. If neither task nor header has the attribute set then, by default, the attribute is set at header level and applies to all tasks. Setting this option will update only the task level attribute used in capability checks, user neons and displayed in the task card – however it will not become visible at the header level.

Allow updates whilst open – normally PSA will not sync with the central db while open – but for some users the list remains open for long periods – this allows synchronisation at the cost of replotting the list and possibly losing position within the list

First 3 materials reserved for substrate – normally selecting material 1, 2, 3, etc displays substrate followed by ink or other detail – this allows only substrate to be displayed against the first 3 materials

Use substrate from any task – most print tasks will have allocated substrate but with a sheeter step the substrate will appear there – this allows the print step to show the substrate for better task sequencing

Allow wide columns in Dynamic Work List Print – in the original version long descriptions were truncated when printed – this uses a modified report to recover the extra text and also allows point size to be reduced giving more columns on the page.

Print Width – there have been problems when printing on a printer that has different default print width resulting in truncation – if there no problems leave at 0 – for A4 landscape set to 16354 – for Letter landscape try 15362 – for Legal try 19602.
