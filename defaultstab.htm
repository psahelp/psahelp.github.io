Screen by screen descriptions > Planning Board Preferences

Defaults tab

Selecting this tab allows column and field select in several places. These represent the Dynamic Work List, Planning Board Task Card, The Planning Board Status Bar, the Planning Board Task Ticket and the Multipart pop up card if Items are in use. The top dropdown selects the area. The second dropdown selects a process type specific list or the default for the area if left blank.The first three have identical source lists that can contain over 200 items if materials, items, resources and notes are all defined.
Up to thirty items may be selected for the task cards and five for the status bar, 20 for the task ticket and 70 for the dynamic work list.

At the bottom left of this screen the width of the title and text areas of the pop up task card may be adjusted. Hover the mouse over the right end of the cells until the ßà arrow appears and drag. Note that in the status bar column widths may also be adjusted to match the entries selected. Use the mouse to drag cell dividers to get the required cell widths.

For the multipart pop up card the default is to show parts across the top and attributes down the left side. This can be reversed by checking the ‘Parts on left’ box.

Selected entries are set for each user and are remembered from session to session.

Recent additions to the options include 'Task Duration', 'Original Work Centre' - if it has been changed by dragging and dropping or capability group scheduling, 'Form Number' and 'Number Up'. The list is regularly extended

‘Select all text of date fields on entry’ - useful for users who prefer to retype dates rather than edit or use the +/- keys to adjust dates

‘Monitor PsaConnect for problems’ – should be turned on by at least one scheduler.  If PsaConnect has to shut down then a message will be created that should be picked up by a scheduler and the cause of the shut down investigated.

‘Check for redraw when switching to planning board’ – normally checked – when the planning board gains focus it will replot if changes have been made since it was last displayed. If replotting take a significant length of time and the mode of working involves a lot of switching to and from the planning board then uncheck this option. It is always possible to replot the planning board by pressing the ‘replot’ button or selecting Display/Replot from the menu.

‘Show maintenance description on task ticket’ – normally checked – maintenance tasks can be set up for any work centre – however they all use the standard order number ‘MAINT’ and this will normally appear on the task ticket. Checking this option replaces ‘MAINT’ with the more informative task description.

‘Use Promised date in place of Delivery date where appropriate’ – normally checked – In PSA on time or late status is derived from the delivery date and the latest possible task start time to meet that date. Checking this option replaces the delivery date with the promised date and should normally be selected.

‘Suppress replacement order messages’ – normally unchecked - after an update ‘What-If’ users receive a warning of the number of outstanding replacement orders in the system. Checking this option prevents the warning ever appearing for this user

‘Suppress ‘Radius deleted’ order messages’ – normally unchecked - after an update ‘What-If’ users receive a warning of the number of any orders in the system that are deleted from Radius or flagged for deletion in Radius. It is essential that these are investigated but the message can be suppressed for some users. Checking this option prevents the warning ever appearing for this user.

‘Suppress replot after drag drop’ – normally unchecked – if replotting takes a significant time and many drag drop functions are performed the replot may be suppressed as for the ‘check for redraw..’ option above.

‘Enable option for scheduled orders only’ - normally unchecked – this is a special – it turns the Schedule Control option to filter for ‘unscheduled only’ orders into an option to filter for ‘scheduled only’ orders

Minimum background processing – suppresses some non urgent housekeeping processes

Parts on left – item/parts pop up will display with item codes down left side rather than across the top – recommended when tasks have multiple items

Items in print change sequence – by default items tend to display in alphabetical sequence – this forces printchange sequence that normally matches jobline from Radius

Show attributes from tasks only – by default if a task has no attribute value the common value from the header will be displayed – this option ensures blank values are displayed

Don’t refresh Dynamic List after Task or Order Maintenance – saves time if lots of changes are made this way and it takes time to rebuild the list

Replace material code with description in pop up and Dups – at some sites the material description is normally used rather than the code
