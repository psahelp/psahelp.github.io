Resources and Key Dates

Resources

All resources can be applied at Order, Component or task level. When applied at component level the resource will apply to all the tasks of the component. When applied at order level the resource will apply to all the tasks of the order. This is typically used for artwork and other approvals that determine whether any work should commence on the order. Which resources apply to which task is determined by the method in use.

If the method in use is changed it is essential to run a full Assessment Check from the planning board and update the live plan with results.

Also on the Functionality tab of the Configuration/System menu is the field Resource Planning – Last Delivery Time. When a resource is marked as expected on a given day any tasks using that resource will be coloured red before the last delivery time and yellow thereafter.

Each of the 30 definable resources can now be set to default to available or not available as required.

User update permission of resources can now be set at individual resource level (a new tab on the user maintenance screen allows definition).

Resources are automatically linked to tasks using the defined ‘dependant process type’ as before. However if this field is left blank the resource becomes ‘global’ and applies to all tasks of the order. Changing this resource value on one task will update all tasks of the order.
There are a number of possible uses of such global resources including a ‘Don’t schedule before..’ resource. If this resource is set to ‘N’ with a date some weeks in the future and resource constrained scheduling is used it will automatically be scheduled starting on the selected date.

Resources declared global can be maintained at the order header level. Global and task level resources can be maintained at component and task levels.

When a resource is marked available ‘Y’ the date and time of this status change is recorded in the ‘Achieved’ column. The available date column has been renamed ‘Expected’ for clarity.
