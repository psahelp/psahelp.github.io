Group Tasks

If task and order grouping has been turned on then grouping a sequence of tasks becomes possible from this list. The Group Tasks screen will open with an allocated group number. On return the group will have been replaced by the group task that may be manipulated exactly as a single task. Work Centre reports and transfers to SFDC still happen as single tasks

To group tasks on a work center they must first be consecutive. Open the dynamic work to list, select the records you are wanting to group and right click a column header - then select the Group option.

 



This will open the Group screen.

 

If you highlight a range of tasks the attribute table will show you which attributes match and which do not

 

Under ‘Group Details’ you can specify Serial – the default or Parallel grouping. Serial grouping is where each task will still run sequentially – you can also specify a percentage make ready reduction that will be applied to all tasks except the first. Parallel grouping is a bit specialised – it is where say 3 tasks are run side by side on the same web requiring appropriate plates. In this case the make ready is the average from the grouped tasks. Press ‘Group’ to link the tasks and close.

The planning board, dynamic work to list and work to list report will all now show the group rather than the underlying tasks. This task may be shunted, dragged and dropped from work center to work center or moved up and down the dynamic work to list in exactly the same way as a normal task. The underlying tasks track the group with appropriate start and end times. If a make ready reduction has been applied the last underlying task will ‘end’ after the group.

If you right click the group task on the planning board or dynamic work to list and select Task Maintenance you will return to the Group Tasks screen. You may select a single or multiple tasks and Ungroup. Once down to a single task the group is destroyed – it is not allowed to have a group containing only a single task.
