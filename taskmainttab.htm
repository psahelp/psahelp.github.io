Screen by screen descriptions > Task Maintenance

Task Maint tab

Make Ready Time - A make ready time can be set against each task.

Task Run Time - A run time can be set against each task.

Overlap Style & Time -The overlap style determines when the following task of the same order starts. Two styles are available the default is Serial, here the following task will start directly after the current task has finished, any move time will be taken into account. If Parallel overlap style is selected the following task may run in parallel to the current task. A delay to the start may be introduced called the overlap time. A check is made that tasks running in parallel cannot ‘overtake’ the previous task.

Move Time - A delay period can be entered during which any following tasks of the same component or dependent components are not allowed to start. For example this may be used to reflect drying or curing times.

Delivery Date and Time – can be set for each task – useful only where a delivery schedule extends beyond the original order delivery date. Tasks with a delivery date greater than the order delivery date will be backward scheduled from the task delivery date

M/R Quantity – from estimating – may be changed

Task Quantity – from estimating – may be changed but will not affect material requirements

Speed Unit/Hr – from estimating – may be changed and will affect run time

Waste % - from estimating – this quantity is included in the task quantity and run time calculation

Held and Reason -

Task Sequence - The task sequence is the order in which tasks of the same component are scheduled – where split tasks or consecutive tasks of the same order on the same work centre are resequenced the task sequence number is used to remember the correct sequence in subsequent scheduling

Number Up – from estimating - the number of items or pages per impression

Form Number – essentially the sheet number – where there is more than one sheet this will be zero

Step Number – this is the costing step number and for Radius II systems is usually the same as the task number. If a task is split both parts will have the same step number so that costs are correctly accumulated. Tasks added in PSA have a zero step number

Original Number Up – Only used where the number up has been changed – this can only apply to web processes in Business Forms

Original Work Centre – records the work centre from estimating so that this can be highlighted during drag drop processes. Also vital if the task is moved to the park area where speeds and times are returned to their original values and a subsequent move to another work centre is then treated as a move from the original work centre to the new.

Ignore Capability Scheduling – A flag that can be set so that this task – probably for some technical reasons – will never be capability group scheduled onto any other work centre

MR Assistants – records the number of Make Ready Assistants used on the task for labour planning purposes

Run Assistants – records the number of Run Time Assistants used on the task for labour planning purposes

At the bottom of this tab brief details of the prior and following tasks are shown

Browse options allow you to move to the prior or following tasks of the order or on the work centre
