Housekeeping

Every day when the schedule is opened for the first time by a ‘What-If’ user a number of automatic housekeeping processes are run. By running them on a daily basis the process will normally only take a second or two. Apart from non working periods – see Non Working Period Automation below – other configuration options are defined under Configuration/System on the right hand side of the ‘File Paths’ tab.

Days to retain completed orders. This has a default of 90 days but it is recommended to set this to a lower value if possible. The maximum value is 365 days. When an order is marked as complete the date is recorded. When this date is more than ‘retain days’ in the past the order will be deleted. Retaining orders for an excessive time will impact PSA performance.

Compact database every n days. This has a default of 30 days but it is probably desirable to set it to a lower value – say 7 days. Because of the volatile nature of a PSA database it should be reorganised and compacted on a regular basis.

Days to retain non working periods. Default value is 10 days, maximum 90 days. Setting this to a low value of 1 or 2 days is recommended. Retaining them serves no useful purpose.

The housekeeping process then checks to see if automatic patterns need to be extended.

Days to retain audit files. Default value is 10 days, maximum 9999 days. Setting this to 10 to 14 days is usually sufficient for resolving problems. Unless they are being analysed for other reasons retaining them serves no useful purpose.

A number of data consistency checks are also run.
