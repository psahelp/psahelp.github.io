User Defined Neons

User defined neons may be created – limited only by space providing a feature that is both simple to use and powerful.

In essence a user defined neon is created at the bottom left of the planning board with a number of simple rules behind it that control both how the tasks on the visible planning board are coloured and a legend to explain the colours. From release 356 these may also be added to Dynamic Work to List.

To set up a sample neon follow these notes – setting your own up after that should be straightforward.

On the User Neon tab enter a 2 character code for the first neon – say ‘CC’ – for customer. Enter a more detailed description to appear on the Tool Tip Text – say ‘Top 5 customer orders’, then set the Active column to Y.

Click below Field in the lower grid – select customer name, in the Test column select ‘=’, in the value column select your top customer – if the planning board is open during this process all the values currently displayed on the planning board for customer will be in the drop down but you can still add other values including wild cards – see table below. Then in the colour column select any of 16 million available colours. The legend column has been added and values you enter here will be used in the Dynamic Work to List if this neon is selected. Repeat for other customers in your list. In the legend area the first five values will appear – you can change the legend if the test value does not make the meaning clear. Apply the change and update the plan. After closing and re-opening the planning board this new neon becomes available.
If setting a neon against a resource there are 6 possible values that you may want to check – these are
A - not applicable - the resource does not apply to tasks of this process type
Y – available
N - not available - no date set
P - Pending - a future date has been set that is before the scheduled start
O - Overdue - a date has been set that is now in the past
T - Too late - a date has been set that is after the scheduled start

When a User Neon is clicked the planning board is coloured by testing each rule until a condition is satisfied. So it is possible to set up complex conditions by careful choice of rules. For example rule 1 is Order Type <> L (Live) colour white. Rule 2 is order quantity < 100000 colour yellow and rule 3 is order quantity >= 100000. This will highlight only tentative orders and divide them into less and greater than 100,000.

To select a neon for the Dynamic Work to List go to Planning Board Preferences, Configure and then select the Defaults or a process specific tab. The user defined neon will be at the end of the list.
