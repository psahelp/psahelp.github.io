Screen by screen descriptions > User Maintenance

Details tab

All current user password entries are listed, new and existing records can be added or changed from here.

Note: - before a password can become operational, amendments must be saved back to the live plan database.

User Name – the user name as it will be displayed at the bottom of the screen and will also appear in audit logs

Op Code – required only if SFDC Lite is in user – must match the users SFDC operator code as it will be used in many records

Password – up to 8 alphanumeric characters – this is stored in encrypted format

Enabled – leave this unchecked to temporarily disable a user

Access Level – select the level required – note that it is recommended to use ‘Limited Access’ and add permissions rather than ‘Fully Amendable’ and try to restrict permission.

Login Message – A message entered here will be shown to the user as they log on
