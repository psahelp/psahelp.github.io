Screen by screen descriptions

Log on

When logging on you have to supply a valid user name and password. This is a recent change – previously only the password was required. However for future log ons the last used user name is remembered so again only the password is required. The user name determines the access level and all permissions within PSA.

The first 4 fields are logon details to the SQL database. If not using Windows Authentication a user name and password are required. These details are stored in encrypted format so that they do not need to be supplied every time

When logging on subsequently you will only require a password.

A post logon welcome message may be displayed. For ‘What-If’’ users they will also receive warnings of replacement orders and ‘orders deleted from Radius’. The latter are vital and must be investigated – see problems section.
