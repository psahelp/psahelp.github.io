Screen by screen descriptions > Order Maintenance

Order Header tab

The order header contains all order specific information. When creating a new order compulsory entries include order number, description, customer and delivery date. The coloured box on the top right hand side of the form denotes the order’s key colour; this can be changed by clicking on the box. The order’s tasks will be displayed in this colour on the planning board. 250 characters of freeform text is available lower left and the 20 technical attributes - formerly shown on a separate tab are shown lower right.
Production Order – this is the order number from Radius – if the order has been reimported then this will have at ‘@’ suffix
Estimate number – transferred from Radius
Master Estimate – transferred from Radius if available
Planning Status - The ‘Plan Status’ of the order has four different states, all new orders have an ‘Unplanned’ status and will not be visible on the planning board. Orders should be ‘Planned’ using the ‘Schedule Sequence Manager’ which handles all the planning processes and allows orders to be split by quantity if required.
Note: Orders will not become visible on the planning board until the order has been scheduled.
Orders can be given a ‘Firm’ plan status if required which prevents scheduled tasks being moved, finally the ‘Completed’ status can be selected once all tasks have been completed. Individual tasks can be given a completed status in the task maintenance screen. Completed tasks can be displayed on the planning board if required through the ‘display options’ facility, and are transparent’.
Description – transferred from Radius
Customer - transferred from Radius – may have an associated colour
Quantity – full order quantity – usually in items
Order Type - An order can be assigned to a particular type either ‘Live’ or ‘Tentative’. If the Tentative type is selected then, when the order is first scheduled, it is given a tentative date and time corresponding to the date and time the order was first scheduled.
Delivery Date - The delivery date should represent the date when the order must be ready for despatch by, the backward scheduling facility uses the order delivery date as a default when scheduling tasks. The delivery date is also used during the ‘Late Orders’ display mode on the planning board.
If the delivery date is changed, the system will automatically conduct an assessment check which calculates the latest possible start date for each task in relation to the order delivery date. A full assessment check on all orders can be conducted from within the planning board.
Individual tasks may be given a delivery date which is independent of their order delivery date, this will have a default set to the order delivery date.
Promised Date - A secondary delivery type date has been provided which is not compulsory, but can be used as a reminder, orders may be sequenced by their promised date in ‘Schedule Control’.
Schedule Date – this is the date the last task is scheduled to complete.
Tentative Date – this is the date a tentative order was first scheduled.
Analysis Codes – three analysis codes with their titles are displayed across the top part of this screen
Minimum Finish Quantity – For very large orders that are likely to be split into several finishing processes a minimum quantity may be entered here. When running the assessment check the latest start time of the final task will be based on completing just this minimum quantity prior to the delivery date. Defining print, Cut and Crease and Glue as all parallel allows a reasonable start time to be calculated.
Technical Attributes - Descriptive text can be entered against each of the twenty technical attribute headers configured in the component type’s screen.
Additional Details – Maintainable fields also show Security status, Press side approval, Category (if the category is rework the original sales order will be show if available. Flags for Invoice Working copy, Proforma Invoice, Consignment order and Individual Packing Slip also appear. These fields are maintainable within PSA and will be copied from Radius in version 1.35 and above.
Notes – a 250 character area is available for order level notes – depending on transfer options this can contain ‘Green Pencil’ detail.
Linked Order Details – if the order is linked to another order in this plan or at another plant brief details of the linked order are shown here

Order Copy Button

The order copy button allows the component and task structure of a selected order to be copied to a newly created order.

The current order is used as a ‘template’, new order header details must be supplied, including order number, description, quantity and delivery date, the customer defaults to that of the ‘template’ order as do all technical attributes. Resource details are not copied

If the ‘Work Centre Make Ready’ check box is checked than all tasks will have their make ready time set to the corresponding work centre default make ready times, instead of using the template task make ready times.

Order Number Change Button

The order number change button allows the order number of the selected order to be changed to a new number. Confirmation is required.
