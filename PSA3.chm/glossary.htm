Appendices including technical notes

Glossary

Assessment Check       Calculates the correct scheduling sequence based on task process types and sequence and component structure. It also calcuates the latest start time for each task with respect to the delivery date, promised date and resource availability status.

Auto-Schedule    Available from the Planning Board this facility will Schedule all Planned orders according to their scheduling sequence

Attribute            Attributes or Technical Attributes can be assigned to orders to describe individual order characteristics

Backward Schedule    Order will be scheduled backwards starting from its  delivery date, last task first

Capability Group Used to describe groups into which Work Centres with similar capabilities can be grouped

Capability Group Schedule  When scheduling tasks may be moved to alternative work centres having similar capability to achieve load balancing among work centres

Component         An order may have several Components usually describing an integral part of the main order e.g. Box,Lid or Insert

Comp’ Relationship     The Planner may define a relationship between the components of an order, to determine the sequence in which they are processed

Component Structure The precedence between components of an order, used to ensure that tasks are scheduled in a logical sequence

Default Order     Used where Radius does not supply task detail when orders are created without estimates

Display Code      Display codes contain user defined groups of machines which can be selected when accessing the Planning Board

Duplicate Order  When a Radius order is changed it may be re-imported to PSA as a duplicate with'@' added to the order number. Original and duplicate are then compared, one selected and the other rejected. A third option now allows the duplicate order to replace the original but continue to use the original scheduled time slots

Extend Shifts      Shifts - also known as non working periods -can be extended forward in time or across work centres

Firm/Free          An order with a Planning Status of Firm cannot be rescheduled

Fixed Point Schedule  An order or group of orders can be scheduled with respect to a specific task

Forward Schedule      Orders scheduled from specific start date, starting with first task

Horizon              The time window duration the Planning Board displays

Infinite Scheduling     Treats each work centre as having infinite capacity. Useful for quickly getting all tasks onto the planning board and outwork processes

Make Ready Time       Calculated length of make ready portion of task

Manual Overloading    Allows Tasks to be placed on top of another task during drag drop operations

Move Time         A delay period during which any following tasks of the same component or dependent components are not allowed to start. For example this may be used to reflect drying time on gluing machines

Non Working Period   Time period attached to a work centre when tasks are not scheduled. Tasks will either be moved to start after the period or extended over it

OverLap Report  The report details any scheduling overlaps between individual tasks (for a given order) across work centres

Overlap style      Determines when the following task of the same order starts.
Serial overlap style - the following task will start directly after the current task has finished, any move time will be taken into account
Parallel overlap style - the following task may start in parallel to the current task, a delay may be introduced called the overlap time

Overlap Time      The delay from the start of one task to the start of another that can run in parallel

Overloading        If ‘Manual Overloading’ has been switched on (default) then tasks can be placed on top of one another

Planned              An order must be Planned before it can be scheduled, this is done using the Schedule Sequence Manager

Planning Board   This is used to display scheduled orders and non working periods

Planning Status  Describes the Planning status of an order or task, these can be Complete, Firm, Planned and Unplanned

Process Type      Describes a specific process used in the production of an order

Promised Date   Promised Have Ready Date

PsaConnect        Background transfer system for transferring data from Radius to Psa and onward to the SFDC system if installed. Data is transferred back updating for example scheduled dates

Resources          The system will allow resource availability planning, available resources are allocated to order components

Resource Update Mode      Available from within the Planning Board when selected a Planner may update resource information relevant to specific orders

Run Status         Describes the Running status of a task once it has been scheduled. These include Make Ready ,Started ,Running, Stopped and Complete

Run Time           Calculated length of run portion of task

Schedule Control        Specific orders must be Planned here prior to scheduling, they can also be placed in a specific sequence for Autoscheduling

Schedule Date    The date and time that the last task of an order is scheduled to finish

Shift method      Describes how tasks dropped on or near non working periods will behave

Shifts                 Shifts are non working periods

Speed                Refers to the Machine Speed of a particular Work Centre in units per hour

Spotlight            Enables specific orders to be located within the Planning Board

Task Card           Information window showing order details upon mouse movement over tasks on the Planning Board

Task Update Mode      Available on the Planning Board, when selected allows the Planner to amend individual task details

Technical Attribute     Up to 20 attributes passed through from estimating. Used for information only other than allowing orders with similar attributes to be planned in sequence. These can now be used to match more accurately work centre 'capabilities' to the requirements of an order

Violation Warning      Warning generated when a task is moved on the planning board. May warn of overlaps to other tasks or resource problems

Wait Time          Period after the end of one task before the next will be scheduled to start. Can be due to drying or curing time or an allowance for moving work from one centre to another

Work Centre             Work Centres hold details of specific machines on which various factory processes are undertaken
