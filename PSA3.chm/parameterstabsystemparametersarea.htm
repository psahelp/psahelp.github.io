Parameters tab - System parameters area

Shift-method defines the way in which tasks placed on the planning board react to non-working periods. With method 1 selected tasks placed in the vicinity of a shift will be automatically be moved to start after the shift is completed, with method 2 the task will be ‘stretched’ over the shift period to compensate for the additional down time. This should be the normal default

If Drag/Drop Style is set to 'Respect non working periods', then tasks placed on the planning board will behave as previously described, however if  this is set to 'Ignore non working periods', than all non-working periods will be ignored and the task left exactly where dropped.

Work centre sequence – where work centres appear in drop down lists this allows the sequence in which they appear to be controlled. This is because some users work with the description and others with the code or process type
