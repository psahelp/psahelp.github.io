Warnings

When dragging and dropping tasks on the planning board there are 10 possible warning messages that can be generated. Here you configure the company wide lowest level of warning (non overridable) that corresponds to the planning board warning button being pressed. For example if moving a task to a work centre of a different process type the move will be prevented and cannot be overridden.

When a warning appears on the planning board there is a further configuration where you set the desired local or higher level of warning. The toolbar button toggles between these two levels.
Note that no warnings are now given about overlaps if the task has been moved to reduce the overlap.

With the 'Violations Warning' button up the local level warnings apply. With the 'Violations Warning' button down the company wide settings apply. When 'configuring' the local settings you cannot set local values to lower levels of validation than the company settings.
The current system is completely customisable between maximum and minimum levels of validation. This allows you to set company wide minimum levels (It is recommended to 'prevent' change of process type unless there is a very good reason why it is OK) and the user can set their own local values to match how they work - getting warning when they want them and not getting them when not needed (say default work centre speed is different)

The planning board warnings are now fully configurable and have some minor changes in detail.

The toolbar button toggles between these two levels. A further message has been added warning of different work centre speeds when dragging tasks between work centres.  No warnings are now given about overlaps if the task has been moved to reduce the overlap.
