Before you start > Upgrading from PSA2

Requirements

At this time the SQL version of PSA will only work with Radius II systems – if you are not running Radius II (any version) you will not be able to link to the new PSA3

You must already have a reasonably current version of PSA2 – i.e. release 380 or better. If this is not the case then first upgrade your PSA2 system to at least this release level.

If the PSA2 option to increase the number of attributes has been applied you now can convert to PSA3.

Check version by opening PSA – the version and release details are at the bottom in the menu task bar

Example Version 2 Release 428 – 6 January 2011

PsaConnect3 version is not critical.

If converting multiple plants into the same SQL database you must make sure that all have the same number of technical attributes – 20 or 34 or 48 – changing all to the highest number is not likely to cause problems unless very large numbers of resources (>20) are in use.
