Process Types, Hold Reasons, Credit Check messages

·    Process types

A process type describes a factory operation such as printing, cut and crease or gluing. A sequence number must also be supplied, operations early on the production process should be given low numbers, and these may be provided in the user choice of increment. Increments of 10 are sensible so that additional process types can be inserted later.

You can also add custom colours for work centres on the planning board to replace the default dark grey (amendable) and white (not amendable).

·    Hold Reasons

When an order or task is held a separate ‘Hold reason’ may be entered. This can be free format or predefined here for consistency. Where predefined reasons are being used user-defined neon can be set up to identify tasks held for each reason.

·    Credit Check messages

With systems linked to Radius II a credit checking facility is available. This is run by PsaConnect and checks the Radius system for order that have exceeded the customer credit limit or where a customer has been put on credit hold. The former will affect a single order; the latter will affect all orders for the customer.

Default ‘Hold’ messages are supplied but these can be overridden on this screen.

·    Approval Status

Some Radius II customers use an item approval system based on tables PV_ItemApproval, PV_ItemApprovStatus and PV_ItemApprovLevel. If this is the case it is essential that at least one level is marked here as PSA approved
