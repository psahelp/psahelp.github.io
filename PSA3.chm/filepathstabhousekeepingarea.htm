File paths tab – Housekeeping area

Days to retain completed orders. This has a default of 90 days but it is recommended to set this in the range 20 - 30 if possible. The maximum value is 365 days. When an order is marked as complete the date is recorded. When this date is more than ‘retain days’ in the past the order will be deleted. Retaining orders for an excessive time will impact PSA performance.

Compact database every n days. This has a default of 30 days but it is probably desirable to set it to a lower value – say 7 days. Because of the volatile nature of a PSA database it should be reorganised and compacted on a regular basis.

Days to retain non working periods. Default value is 10 days, maximum 90 days. Setting this to a low value of 1 or 2 days is recommended. Retaining them serves no useful purpose.

Days to retain audit files. Default value is 10 days, maximum 9999 days. We now recommend setting this to around 30 days. This is usually sufficient for investigating problems. Unless they are being analysed for other reasons retaining them serves no useful purpose.

Report Format allows users to select between A4 and Letter paper size for production of all reports. The reports will be printed in the selected language.

Completion Report – as each task completes PSA can run a ‘completion report’ that acts as a snapshot of details at the moment of completion. The report to run should be defined under work centre maintenance in the ‘Completion Report’ box. If desired this report can be written direct to disk in the background rather than printed.
The directory is defined here and the file name to be used can contain these variables - %O – order number, %C – component number, %T – task number, %W – work centre, YYYY – year of completion, MM – month of completion, DD – day of completion. The extension should be set to doc, xls or pdf for word, excel or pdf format outputs. Therefore a typical file name might be %O_%T_YYYYMMDD.pdf
