Preferences tab

On the Preference tab you will find 6 separate configuration areas.

·    Planned/unplanned columns

In the planned column/unplanned column area you select the columns and column sequence to be use for the unplanned and planned orders tabs. Choose from 30 available fields plus all your user defined technical attributes.
The task time, task quantity, from week and to week columns only apply if a work centre or work centres (by display code) are being ‘monitored’ – see below under Utilisation Monitor for details.

·    Additional Filters

Three additional filters by technical attribute can be added to the planned and unplanned tab – select the attributes here

·    Schedule Colour Boundaries

On the planned order tab are 3 fixed columns headed ‘Before’, ‘After’ and ‘Order’. The Before and After columns show one of 5 colours if the order is scheduled. These indicate where the scheduled completion date lies in comparison to the delivery date or promised date if specified. This calculation falls into one of five colour bands and in this section you define the boundaries between each band. Obviously the first and last bands are open ended – allowing for exceptionally early to exceptionally late. The numbers selected should progress from left to right. If two adjacent boundaries are declared the same no orders can appear in the band between. If all boundaries are set to 0 hours all orders will appear either red or green.

·    Initial contingency times

During the schedule process each task is laid down following a preceding task in a parallel or serial style using ‘move time’ (time defined after one task before the following task can start to allow for drying, curing or even moving), ‘wait time’ (minimum run time of first task before second, parallel, task can be started.
This resulted in all tasks usually being scheduled in a very short time frame.
Contingency time adds a flexible extension to this process. When scheduled (forward or backward) the tasks are laid down adding ‘contingency’ time between the tasks and after the last task prior to delivery. If the order is still late then the schedule sequence can be changed or some other action taken.
Note that this leaves ‘move time’ to be used for its correct function of spacing tasks. The scheduling process will always respect ‘move time’.

Contingency time can now be added to both forward and backward scheduling processes. This will normally result in task placement that allows individual tasks to be moved forwards or backwards in time without overlapping other tasks of the same order. Previously this could only be achieved by adding an artificial and inflexible 'wait time' to each task.

Between tasks:-Specified in hours and minutes this delay will be added between tasks as the order progresses through the plant. For multiple passes on the same work centre no time is added.

Before Delivery:-Again specified in hours and minutes this is the minimum time before delivery that the last task should complete.

In Action:-During the scheduling process PSA will attempt to leave the required contingencies between tasks and prior to delivery. However if there is insufficient time then new, reduced contingency values are calculated and the order is scheduled again. If the delivery date is very close this process can repeat 2 or 3 times after which the contingencies are set to zero and the earliest possible delivery is calculated.

Initial Values:-Sensible initial values will vary depending on the type of work and average length of task and you should experiment to find the best values for each plan. It is recommended that the first values used are in the range of 12 - 24 hours between tasks and 24 - 48 hours prior to delivery.

·    Utilisation Monitor

We are going to calculate the time and quantity of tasks in the system for one or more work centres – these are defined by a display code – either here or on the Planned tab

You can select the analysis period as months, weeks, days or shifts – fixed at 8 hours – but you can offset these from midnight – if today’s shifts are 6-14, 14-22 and 22-6 then mark the offset as 06:00. If the first shift starts at 22:00 last night mark the offset as -2:00
