Shop Floor

The upper part of this tab shows information returned by SFDC including all quantities and percentage complete and waste. Also average speeds, total elapsed run time and number of stops. None of this is maintainable – SFDC has control

The lower part of the screen allows manual task updating that allows the planner to simulate individual task progress on the shop floor. On toggling the relevant toolbar button  tasks shown on the planning board are coloured according to their current running status.

Only one task per work centre can be set as running at any time, the SF neon can be used to search for running tasks. If the user selects a task on a work centre that already has a running tasks a warning is displayed and an option button allows the user to move directly to the running task.
The only exception to this rule is for outwork work centres. These are recognised by the code starting ‘O/W’. For these centres any number of tasks can be marked as running. The recommended method is to ‘start’ tasks when they are sent out – the running status indicating those that have been sent out. On return they are marked as complete and drop from the board.

Double clicking on specific tasks in this mode will load the ‘Manual Task Update’ screen; this allows the planner to place the task in the required running status. The task will normally move to start at the current date and time shown as ‘Update Date/Time’, if the ‘Schedule Start’ check box is checked then the task will remain at its current schedule start date.

A warning message has been introduced to the Manual Task Update routine. If the quantity entered as complete exceeds the task quantity plus an allowed percentage of overs a warning is displayed.

Selecting OK confirms the entry, Cancel escapes and prompts for re-entry of the correct value.

The limits are set and warnings activated within work centre maintenance.

The quantity completed can also be updated provided the task is ‘Running’. Changes to the quantity completed will cause the run time be adjusted accordingly, which in turn causes the length of the task to adjust as shown on the planning board.

If the ‘Shop Floor Feedback’ check box is checked then both running status and quantity completed will be set using data fed back from the shop floor data collection system. This requires a PsaConnect communication link to be in place.

If the 'Scheduled Start option is checked then the task will display at the original point in the plan with a length reflecting the remaining run time. If not checked then the task will display at the Update date/time with the same length. The previous setting of this check box is remembered so you will not normally need to set or unset its value.
