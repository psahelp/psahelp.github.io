Buttons

 Reload Button

Task information can be changed by the planner by many modes of operation, from dragging and dropping single tasks to scheduling entire orders. Any changes are made locally on the individual user’s machine. The user can save his/her changes back to the Live Plan if required (see Updating Live Plan) at any time, or the user can produce ‘what if’ scheduling operations and manipulate data to experiment with the schedule. If the user wishes to return to the current live plan, to the point at which it was last updated then the ‘Reload Live Plan’ option  can be used, returning all tasks back to their original positions.

·    Updating the Plan

All changes to data all conducted locally on the individual user’s machine. If the changes made are to be retained the live plan database must be updated. Once this has been done changes are then available to other users. The ‘Update Live Plan’ screen can be accessed by pressing the  button from the planning board and is also available from the ‘File’ menu.

The user can enter a brief update message if required, the entry will default to the users user name, pressing the ‘OK’ button will initiate the update process.

The status of the live plan is immediately visible together with the last 10 update messages. The box lower left will show red if other users have changed the plan since you opened your copy. If it shows green then the plan has not been changed by any other users and updates can be applied without problem. If the ‘Reload Live Plan’ check box is checked then the newly updated plan is copied back onto the user’s machine and re-opened which will then hold both the current users changes and changes made by other users since loading.

The system will automatically perform validation checks, and will report any changes which can not be made; this may result if the user attempts to update records which have already been updated by another user since first opening the plan. Details of these can be seen in the ‘Update Log’ accessed by pressing the ‘Log’ button; they can also be printed using the ‘Print’ button.

Validation checks can be by-passed if the ‘Override Validation’ box is checked, in which case all recent changes will be passed back to the live plan, regardless of whether changes have already been made to the same records since loading.

If the Planning board is open when the Plan is reloaded then it is automatically replotted to reflect the latest Live Plan position.

·    Print Button

The planning board can be printed by pressing the third toolbar button  from the left. The output will be sized automatically for best fit on the page

·    Replot Button

The  button repositions all the tasks on the planning board in their scheduled positions. This may be useful after dragging and dropping tasks around the screen, hidden tasks will be lifted slightly so that they become visible. The tasks will be coloured according to the current display mode

·    Colour by Order Button

The  button will re-colour all tasks according to their pre-defined order key colour

·    Spotlight Facility

The ‘Spotlight’ facility  allows a specified order to be located anywhere on the planning board, provided at least one work centre has been selected which is utilised by a task of the selected order. The spotlight facility will automatically move to the first task found in the current display code and move the planning board start date in accordance.

The Spotlight facility can also be used to highlight tasks that belong to a particular customer – select from the drop down list.

Where item details have been imported into the plan the planning board may be searched for a selected Item. Tasks are highlighted that have the item attached.

This function is now largely replaced by the more powerful User Defined Neon function

·    Technical Attribute Display

Allows tasks to be ‘highlighted’ that have similar technical attributes, these are maintained against their order headers. A specific process type must be selected together with the required attribute header. Specific attribute data can then be highlighted if required. ·    Task Card Button

When pressed toggles the Task Card display facility. This enables a small green box to appear as the mouse pointer is moved over a task. The Task Card displays up to 10 order or task details selected by the user. This is configured by pressing the 'Task' button on the Planning Board Entry screen.This facility can be switched off by clicking on the button again

·    Warnings Button

When pressed reduces the warnings to the minimum level defined under Configuration/System. Raised you will have most warnings..

·    Drag and Drop

Individual tasks can be dragged and dropped around the planning board as required. The  toolbar button can be used to switch task drag and drop ‘On and Off’.

·    Assistants Button

Turns the assistants display on and off. This display will appear at the bottom of the screen if the horizon is 7 days or less. The number of make ready and run assistants for all the tasks in the current display code are added and displayed as a graph – the yellow band for make ready assistants, the green band for run assistants and the blue band for the total assistants. The peak values for each graph are shown at the right hand end. Assistant numbers can be imported from Radius or manually added via task maintenance

·    Auto infinite fixed point following

When turned dropping a task in a new location will take the following tasks of the same job and move them up on their current work centre to follow the prior task as closely as possible. Typical use is scheduling a task at the end of a long list and then dragging the print task up to the required print date – other tasks will also move up. Normally this is only turned on when required.

·    Linked Drag Drop

This feature, if enabled using the  button, allows tasks of the same order number to be dragged and dropped as if they were one single task.

The task shown being moved on the Domino work centre is comprised of two tasks from the same order. On movement of the leftmost task any following tasks on the same work centre with the same order number will be moved with it.

·    Shunted Drag Drop

With this option  selected a task dropped onto two touching tasks will now start AFTER the first task and not exactly where dropped. Subsequent tasks are shunted back to remove overlaps as before. This feels more natural. If the task remains on the same work centre then the direction of shunting is opposite to the direction of dragging. If a task is dragged to the right then the tasks over which it has been dragged will be shunted back to the left and vice versa. This maintains a sequence of consecutive tasks rather than introducing random gaps.

·    Schedule Control Button

The  button opens the Schedule Control screen to enable the full range of planning and scheduling functions to take place.

·    Fixed Point Schedule

Invokes the fixed point scheduling screen. This facility allows tasks to be scheduled about a specific work centre. For example, if a task is moved forward in time on a specific work centre, it is possible to re-schedule all the following tasks to prevent any overlaps.

 

 

A specific order can be selected otherwise the system will automatically schedule all the following tasks for the orders present on the selected work centre.

·    Shifts Display Button

The  button. This causes all existing shift periods to be displayed. When toggled then the two adjacent buttons will become enabled. These are ‘New Shift’ and ‘Extend Shifts’, both of which are available from the ‘shift maintenance’ menu.

·    Adding New Shifts Button

New non-working periods or shifts can be added by clicking on the  button which will activate the ‘New Shift’ screen.

·    Shop Floor Running Tasks

The SFDC running task bars may be switched on and off by the  button. The  button is used to run the Shop Floor Report. This has been enhanced to allow reporting for a single work centre, all work centres or a 'display code' of work centres. You can create display codes specifically for this purpose.

The shop floor enquiry data is designed to be used with PsaConnect which provides a communication link between the scheduling assistant and the shop floor data collection system (SFDC). Scheduled order information is passed down to the shop floor and running information relating to individual tasks is passed back to the scheduling assistant.

The shop floor enquiry board allows the planner to view the running information relating to individual tasks, and keep track of what is happening on the shop floor. The planning board can be updated with this information to provide a more accurate plan, enabling running tasks to be moved into their actual start and calculated end or completed end time positions, details of running status and quantity completed can also be updated.

The scheduled tasks are coloured, if ‘active’, according to their current running status as signalled from the SFDC system, the key is shown at the top of the screen.

Running Bars - are used to represent the actual start time and calculated end time of individual tasks, the quantity completed is also reflected as a percentage on the running bar.

Running Details - Further details can be shown by double clicking on the selected planned task, which will load the Task Maintenance screen on the SFDC tab. This reveals more detailed quantity and start time information which has been passed directly from the SFDC system by PsaConnect.

·    Polling PsaConnect Button

The live plan update screen has a ‘Poll_SF’ button  which when pressed will cause PsaConnect to be ‘Polled’ if in use. This will allow all recent live plan changes to be passed out to the shop floor system by PsaConnect

·    Order Maintenance Button

When pressed the order maintenance screen will open. If a task has previously been clicked it will open at that particular order – otherwise it will open on the search tab

·    Task Maintenance Button

When pressed the order maintenance screen will open. If a task has previously been clicked it will open at that particular order – otherwise it will open on the search tab

·    Undo Button

An ‘Undo’ button  has been provided on the planning board toolbar, which will move tasks back to their previous positions after a drag/drop event or any ‘shunt’ operation. The facility will not move task’s back after any scheduling operations.
