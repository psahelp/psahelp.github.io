PSA Loader

This appendix applies to sites that use PSA3 installed on individual PCs around the plant. If you are using Citrix or Terminal Services it should be ignored.

The objective is to simplify and make more reliable the upgrading of PSA3 across a large number of separate PCs.

Method

The zip Loader.zip contains a new program PSA_Loader.exe and a small file stp.txtx. Using this new program should transform future upgrades. It works like this.

 a) Each user still requires PSA to be installed on their local PC - all the dlls have to be there and registered.

 b) Somewhere on the server (suggest adding a 'runtime' directory below the PSH database) put the files PSA3.exe, PSA_Loader.exe and stop.txtx.

 c) Change the icon on each users desktop to point to the centrally located ......\PSA_Loader.exe rather than the local PSA3.exe

 d) When they click the icon the loader will run and immediately load the central PSA3.exe and all is normal.

 e) When you want to upgrade - change the name of stop.txtx to stop.txt. Within 5 minutes all AutoSync users and all What-If users without unsaved changes will detect the file and shut down. What-If users with changes will shut down after the next update. As they shut down PSA the loader is relaunched.

 f) The loader will now show 'Waiting for upgrade'.

 g) Once all users have stopped running you can replace the PSA3.exe with the new version. 

h) Last step - rename stop.txt back to stop.txtx - and the launcher will detect the change within 10 seconds and start the new version of PSA.

It is recommended that before the last step you run the new PSA3.exe directly and open the plan to perform any upgrades. This will ensure that only one user - you - attempts to upgrade the database rather than all users trying to do so at the same time.
