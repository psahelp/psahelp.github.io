Task Maintenance

Detailed task information is maintained in the task maintenance screen. When adding new tasks the planner must specify to which component the task belongs so these must be defined first. The task number is generated automatically but can be changed before the new task is saved.

The work centre on which the task is to run must be selected, the task description will default to the work centre name; this can be changed if required. On selecting a work centre the task speed will be set to the default speed of the work centre which in turn will cause the task run time to be calculated (depending upon the task quantity).

Further changes to speed or quantity will also cause the run time to be re-calculated accordingly.

If the run time is changed then the speed will be recalculated.

At the top of this screen the common area shows
Order number and description
Component and description
Task Number and description
Plan Status – unplanned, planned, firm, completed
Planned Start Time – as it will appear on the planning board
Planned End Time – as it will appear on the planning board
Run Status – not started, make ready, run, suspended, complete
Actual Start Time – from SFDC or manual entry
Calculated End Time - from SFDC or manual entry
Work Centre – a drop down list all other work centres of the same process type – all work centre can be selected for the drop down but this is not recommended
Completed Quantity – from SFDC or manual entry
Remaining Quantity– by subtraction from original quantity
