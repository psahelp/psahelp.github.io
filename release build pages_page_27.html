Imported Duplicate Orders
The PsaConnect system that transfers order details from Radius to the Scheduling Database can be used to re-import orders which may have already been imported, planned and scheduled. Any orders of this type will be given a terminator character ‘@’. When attempting to plan orders of this kind the system will automatically detect any differences between the newly imported order and the existing order, the screen below will automatically load and provide details of differences.
Four options are then open to the planner:
  1. Delete New Order, the newly imported order can be deleted using the ‘Delete’ button, this should be considered if the new order has no significant differences to the existing order.
  2. Replace Existing Order, if the new order does have significant changes then it would be sensible to replace the existing order with the newly imported order, this can be done using the ‘Replace’ button, the new order can then be planned in the usual way.
  3. Merge with existing Order. Where the new order has the same tasks on the same work centres then a third option is available. This merges the new order details with the old order schedule times. If task lengths have changed there may be overlaps that will have to be resolved.
  4. Use the 'Match Up' facility. This opens a new window that matches tasks and components where possible leaving extra or missing tasks unmatched. The user can then select or deselect matched pairs by clicking appropriate rows in the left and right tables. On completion the 'Schedule' button is pressed and the scheduled time slots from the old order on the right are used to schedule the replacement on the left. Any unmatched tasks are 'infinite scheduled' onto the planning board at the earliest time. This is likely to result in overlapping tasks but the new order has not lost its position in the schedule queue. This method will work both for changed orders from Radius (with trailing '@') and Live orders replacing tentative placeholders where the order numbers are completely different. The ‘Retain unmatched tasks’ box allows you to retain any new tasks that have been added to the original order. This is quite a specialized function so the checked state is not remembered and it always defaults to unchecked
