Before you start > Upgrading from PSA2

Using PSA3 for the first time

 

PSA3 can be installed into the same directory as PSA2. There are no shared files

Go to the Windows directory and copy PSA2.ini and name it PSA3.ini. Enough parameters are common to make this the best possible starting point.

Now start PSA3.exe – double click or create a desktop icon.

The logon screen will require full logon details for the SQL database. If user names and passwords are required they will be stored in an encrypted format so that repeat entry in not needed. The user and PSA password remain the same and will have been converted.  Enter PSA_SQL as the database and complete the Company and Plant fields. At the moment PSA3 remains single company/plant but the central SQL database can hold multiple company and plant data.

As a local database does not exist PSA will create one from psa3_std.psh and run a full data load – possibly followed by housekeeping processes.
