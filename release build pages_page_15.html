Create a database
Now we need to create a new empty database. Because of possible permissions issues there are no plans to enable this direct from PSA or any convertor. Right click on Databases and select ‘New Database…’. Enter a database name for this – say PSA_SQL – and set the initial size to say 100 Mb. Change the ‘Autogrowth’ to ‘by 10 Mb, unrestricted growth’. You don’t have to do this but it is good practice. Change the database and log file locations – I recommend putting them in the same directory you would normally put the Live.PSH database. This is not essential but it keeps things in familiar directories. If you have test and live databases they can now use use the same directory – there are no conflicting file names.
