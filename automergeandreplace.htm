Auto Merge and Replace

When orders are changed in Radius they are re-imported to PSA as ‘replacement’ orders with a ‘@’ suffix added to the order number. These are always unplanned. During the ‘plan’ step a difference check routine displays the changes to the user and various options to merge or replace are offered. If there are many small updates to an order this process can be annoying so a user definable degree of automation is defined here.

‘Automatically schedule new orders imported as planned’ – PsaConnect allows new orders to have a planned status when imported – this option allows such orders to be scheduled in the background using the default scheduling settings – use with care

‘Use advanced work centre replacement before merging’ – this option must be treated with great care – when a replacement order is received it will determine what happened to the original order and attempt to repeat on the replacement using capability groups and process type to assign tasks to the work centres used by the original order. It will then match-up and schedule in the background. It is very unlikely that any orders will fail the process.

‘Allow automatic merging’ – with this checked replacement orders go through a difference check that creates a list of differences. If there are no differences or those differences have been marked in the list below as ‘trivial’ then the replacement order will be merged with the time slots and work centres of the original order and the original order will cease to exist. This may generate some gaps or overlaps if task quantity changes have reduced or extended run times.

‘Allow automatic replacing’ – similar to automatic merging. If automatic merging has failed then the same list of differences is compared with those in the list below. This now contains differences like ‘extra tasks’ or ‘added items’ that would always prevent merging. If those differences have been marked in the list below as ‘trivial’ then the original order will be replaced and will cease to exist. This leaves the replacement order unscheduled and therefore should again be used with care.
